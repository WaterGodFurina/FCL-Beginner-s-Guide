# 安装包后缀名的区分（手机、Windows 电脑、Linux）

## 为什么不同平台安装包不一样？

不同操作系统（手机 Android、电脑 Windows、电脑 Linux）的**底层架构、文件格式、安装机制**都不同，所以安装包格式也不同。

打个比方：同样是「装一个软件」，安卓手机用 `.apk`，Windows 电脑用 `.exe`，Linux 用 `.deb` 或 `.rpm`——就像同样是「点菜」，中餐厅用中文菜单、法餐厅用法文菜单、日料店用日文菜单，**互相看不懂**。

:::note 关键概念
**安装包不通用**：`.apk` 不能在 Windows 上装，`.exe` 不能在手机上装，`.deb` 不能在 Windows 上装。**不是文件坏了，是平台不对**。
:::

## 三大平台安装包对比表

| 平台 | 主流安装包后缀 | 由谁开发/维护 | 安装方式 |
|------|---------------|---------------|----------|
| **Android（手机）** | `.apk` / `.apks` / `.xapk` / `.aab` | Google | 文件管理器点击 / 应用商店 |
| **Windows（电脑）** | `.exe` / `.msi` | 微软 | 双击运行 |
| **Linux（Debian 系）** | `.deb` | Debian 项目 | `dpkg -i` 或 `apt install` |
| **Linux（Red Hat 系）** | `.rpm` | Red Hat | `rpm -i` 或 `yum install` |
| **macOS（苹果电脑）** | `.dmg` / `.pkg` | 苹果 | 拖拽到 Applications |
| **iOS（iPhone）** | `.ipa` | 苹果 | App Store / 签名工具 |

## 一、Android（手机）—— `.apk` 系列

### `.apk`（Android Package）

最基础的安卓安装包。本质是个 ZIP 压缩包，里面装着：

- `classes.dex`：App 的 Java 代码（编译成 Dalvik 字节码）
- `resources.arsc`：编译后的资源
- `AndroidManifest.xml`：清单文件，声明权限、组件
- `res/`：图片、布局等资源
- `lib/`：`.so` 共享库（不同 CPU 架构各一份）
- `META-INF/`：签名信息

### `.apks`

**拆分版 APK**。Google Play 现在要求开发者上传 `.aab`（Android App Bundle），商店根据用户手机配置，**只下发需要的部分**（比如只下发 arm64 的 `.so`，不下发 x86 的），节省空间。这些拆开的包打成一个 `.apks` 文件，需要用 [Split APKs Installer（SAI）](https://github.com/Aefyr/SAI) 等工具安装。

### `.xapk`

APKPure 网站定义的格式，本质是个 ZIP，里面装着基础 APK + 各种扩展包（OBB 数据、配置文件等）。用 [APKPure 自己的安装器](https://apkpure.net/apk-installer)能装。

### `.aab`

**Android App Bundle**，开发者上传到 Google Play 的格式，**不是直接安装的**。商店会把它拆成多个 split APK 下发给用户。

## 二、Windows（电脑）—— `.exe` / `.msi`

### `.exe`（Executable）

Windows 可执行文件，最常见。双击就能运行，可能是安装程序，也可能是绿色软件（直接能跑，不用装）。

`.exe` 文件本质是 **PE 格式**（Portable Executable），里面是 Windows 能直接执行的机器码。

### `.msi`（Microsoft Installer）

Windows 官方安装包格式，必须通过 Windows Installer 服务安装。比 `.exe` 更规范，能记录安装信息、方便卸载。Office、Visual Studio 等大软件常用。

### 其他

- `.msix` / `.appx`：Windows 10+ 的新格式，类似手机的沙盒应用。
- `.bat` / `.cmd`：批处理脚本，不是安装包但能执行命令。

## 三、Linux —— `.deb` / `.rpm`

Linux 不像 Windows 那样双击就能装，主要用**包管理器**安装。

### `.deb`（Debian）

Debian 系（Ubuntu、Linux Mint、Debian）用的格式。安装命令：

```bash
sudo dpkg -i package.deb
# 或者
sudo apt install ./package.deb
```

### `.rpm`（RPM Package Manager）

Red Hat 系（Fedora、CentOS、openSUSE）用的格式。安装命令：

```bash
sudo rpm -i package.rpm
# 或者
sudo yum install package.rpm
# 或者
sudo dnf install package.rpm
```

### 其他

- `.AppImage`：免安装格式，下载后给执行权限就能跑，类似 Windows 绿色软件。
- `.flatpak` / `.snap`：跨发行版的现代打包格式。
- `.tar.gz`：源码包，要自己编译（`./configure && make && make install`）。

## 四、为什么不能互通？

根本原因：**不同系统的可执行文件格式不同、CPU 指令集不同**。

1. **可执行文件格式不同**：
   - Android 用 `.dex`（Dalvik 字节码）
   - Windows 用 PE 格式
   - Linux 用 ELF 格式
   - macOS 用 Mach-O 格式

2. **CPU 指令集不同**：
   - 手机用 ARM 指令集
   - 老电脑用 x86 指令集（Intel/AMD）
   - 新 Mac 用 ARM 指令集（M1/M2/M3）
   - 一份编译好的机器码只能在对应指令集的 CPU 上跑

3. **系统调用不同**：每个系统的 API 都不一样，Windows 的程序调 Windows API，Linux 的程序调 Linux API，互不通用。

所以你拿一个 `.exe` 想在手机上跑，手机系统看不懂 PE 格式，CPU 也看不懂 x86 指令，**根本跑不起来**。

## 五、跨平台怎么办？

想在不同平台跑同一软件，有几种办法：

### 1. 软件本身跨平台

开发者用 Java、Python、Electron 等跨平台技术写，**同一份代码编译成不同平台的包**。比如 VS Code 有 Windows 版、Mac 版、Linux 版，但都是同一套源码。

### 2. 模拟器

用一个软件模拟另一个平台的环境：

- **[WSL](https://learn.microsoft.com/en-us/windows/wsl/)**（Windows Subsystem for Linux）：Windows 上跑 Linux。
- **Wine**：Linux 上跑 Windows 程序（兼容性有限）。
- **[BlueStacks](https://www.bluestacks.com/) / [雷电模拟器](https://www.ldmnq.com/)**：电脑上跑安卓 App。
- **[PojavLauncher](https://github.com/PojavLauncherTeam/PojavLauncher)**：手机上跑 Minecraft Java 版（后面会讲）。

### 3. 虚拟机

在系统里装一个完整的虚拟系统（[VMware](https://www.vmware.com/)、[VirtualBox](https://www.virtualbox.org/)），里面跑另一个系统。

## 小结

| 平台 | 安装包 | 不能在 |
|------|--------|--------|
| Android | `.apk` / `.apks` / `.xapk` | Windows、Linux、iOS |
| Windows | `.exe` / `.msi` | Android、Linux、macOS |
| Linux (Debian) | `.deb` | Android、Windows、Red Hat |
| Linux (Red Hat) | `.rpm` | Android、Windows、Debian |
| macOS | `.dmg` / `.pkg` | Android、Windows、Linux |
| iOS | `.ipa` | Android、Windows、Linux |

**核心原因**：可执行文件格式、CPU 指令集、系统 API 都不同。
