# 电脑Windows开服

## 1. 安装Java

### 1.1 Java版本挑选

不同Java版本支持的MC范围不同：

- **JDK 8**（2014）：最经典，企业还在大量用，Minecraft 1.7–1.16.5 用这个。
- **JDK 11**（2018）：长期支持版（LTS），不常使用。
- **JDK 17**（2021）：LTS，Minecraft 1.18–1.20.4 要求。
- **JDK 21**（2023）：LTS，Minecraft 1.20.5–1.21.11 要求。
- **JDK 25**（2025）：当前最新 LTS，Minecraft 26.1+ 要求。

### 1.2 下载Java

这里使用[Microsoft Build of OpenJDK](https://learn.microsoft.com/zh-cn/java/openjdk/download)演示
![](/img/docs/开一个属于自己的服务器/开服操作/电脑Windows开服/MicrosoftBuildOfOpenJDKDownloadPage.png)
根据实际所需找到对应版本的表格,找到带有`Windows	X64`的一行,`exe`和`msi`都可以
<img src={`${imgBase}/FindMicrosoftBuildOfOpenJDKOfWindowsX64.png`} />
下载完成后双击,默认点下一步就可以了,如果你想修改安装路径,在这个界面可以手动更改
<img src={`${imgBase}/ModifyJDK_InstallationPath.png`} />

### 1.3 配置环境变量(可选)
如果你是下载的是**Java 8**或者软件**找不到你下载的Java**,那么就需要配置**环境变量**
按下`Win+X`,点击`Windows PowerShell(管理员)`,输入以下命令
<img src={`${imgBase}/WinX.png`} />

```PowerShell
[Environment]::SetEnvironmentVariable("JAVA_HOME", "path", "Machine")
$oldPath = [Environment]::GetEnvironmentVariable("Path", "User")
[Environment]::SetEnvironmentVariable("Path", "%JAVA_HOME%\bin;" + $oldPath, "User")
```

这里的path需要替换为实际路径,默认为`C:\Program Files\Microsoft\jdk-xxx-hotspot`,xxx替换为实际版本,例如:25.0.3,完整的命令则是

```PowerShell
[Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Microsoft\jdk-25.0.3-hotspot", "Machine")
$oldPath = [Environment]::GetEnvironmentVariable("Path", "User")
[Environment]::SetEnvironmentVariable("Path", "%JAVA_HOME%\bin;" + $oldPath, "User")
```

:::tip
Path和path的区别
| Path | path|
|------|------|
| 系统级环境变量 | 在这里指Java安装路径,后面不接反斜杠(\\) |
:::

## 2. 认识服务器核心

| 核心分类 | 常见代表核心 | 支持的版本范围 | 核心优势 | 核心劣势 |
| :--- | :--- | :--- | :--- | :--- |
| **原版核心** | Vanilla | **全版本**（同步官方快照与正式版） | 更新最快，完美还原官方所有新特性与红石机制 | 性能优化极差，**不支持**任何插件和Mod |
| **插件核心** | Paper, Purpur, Spigot | **主流全版本**（多用于1.12.2及以上新版本） | 优化极佳，高版本不卡顿，**玩家不需装任何额外文件** | **不支持Mod**，无法增加游戏本体没有的方块/生物,不过可以使用插件解决 |
| **模组核心** | Fabric, Forge, NeoForge | **特定黄金版本**（如1.7.10, 1.12.2, 1.16.5, 1.18.2, 1.20.1等） | **支持大型Mod**，Fabric在新版本表现好，Forge老生态无敌 | 客户端**必须安装相同Mod**，版本生态断层严重 |
| **混合核心** | Arclight, Mohist | **部分版本**（多集中在1.12.2, 1.16.5, 1.18.2, 1.20.1等Mod活跃版本） | **同时支持Mod和插件**，既有丰富玩法又方便服务器管理 | 兼容性最差，插件与Mod冲突时极易导致服务器崩溃 |

## 3. 使用Minecraft Server Launcher(MSL)启动器开服

<img src={`${imgBase}/StartNewServer.png`} />
点击**自定义模式**，名称和目录按需设置，Java环境可以点击**电脑上的Java**旁的**点我检测**，也可以选择**使用环境变量**和**选择地址**

<img src={`${imgBase}/CustomMode.png`} alt="自定义模式" />

下面的默认，服务器核心/版本按实际选择，点击**下载**、**下一步**，完成后对服务器进行设置

在**实例设置**里把正版验证关了，其他的可参考[此表](window:/term/ServerConfigList.md),拉到底部点击**保存**，服务器启动！

<img src={`${imgBase}/ServerConfig.png`} />
## 4. 古法开服(非遗文化传承人doge)

把下载的服务器核心放到一个空白文件夹，在顶部的路径栏输入`cmd`回车
<img src={`${imgBase}/OpenCMD.png`} />
输入下列命令
```cmd
java -jar ServerCore nogui
echo eula=true>eula.txt
powershell -Command "(Get-Content 'server.properties') -replace 'online-mode=true', 'online-mode=false' | Set-Content 'server.properties' -Encoding utf8"
java -jar ServerCore
echo 'java -jar ServerCore' > start.bat
```

:::tip
这里的`ServerCore`需要替换为实际核心名，需要带上[文件扩展名](window:/term/后缀名.md)，如`paper-1.21.11.jar`
:::

如果想自定义更多功能，可用记事本参考[此表](window:/term/ServerConfigList.md)修改`server.properties`，之后每次启动服务器则只需要运行`start.bat`
